Ladybird Anti-Fingerprinting Engine

Component-level privacy infrastructure for browser internals. A deep dive into building cohesive anti-fingerprinting systems across WebGL, Canvas, Audio APIs, and navigator spoofing.

contents

Overview

Browser fingerprinting represents one of the most sophisticated privacy threats on the modern web. While user agents are easy to spoof, the real attack surface lies in the myriad of APIs that leak implementation details: WebGL renderers, Canvas rasterization behavior, AudioContext sample rates, and thousands of navigator properties.

This project implements a component-level anti-fingerprinting system within the Ladybird browser engine. Rather than applying generic obfuscation globally, the system enforces internal coherence — ensuring that spoofed data remains consistent across all APIs and contexts while maintaining full platform compatibility.

Key Insight

The danger isn't what sites know individually—it's what they can correlate. By generating internally-consistent fake data profiles and enforcing hard boundaries between origin contexts, we make cross-site correlation attacks exponentially harder.

Architecture

Core Design Principles

The system is built on three foundational principles:

1. Origin-Scoped Profiles: Each origin gets a unique, deterministically-derived fake profile that never changes within that origin context.

2. Component Enforcement: Spoofing happens at the component level—WebGL renderers are generated in the graphics engine, Canvas data in the rendering pipeline, Audio properties in the audio subsystem. This prevents API-level inconsistencies.

3. Platform Coherence: All spoofed values must remain compatible with the underlying platform. A Windows 11 system can't suddenly report a macOS GPU vendor.

FingerprintProfile {
  origin_hash: blake2(site_origin),
  gpu_vendor: deterministic_select(platform_compatible_vendors),
  webgl_renderer: synthesize_consistent_renderer(),
  canvas_seed: profile_hash & 0xFF,
  audio_context: {
    sample_rate: select_compatible_rate(),
    channel_count: platform_max_channels(),
    state_buffer: [...]
  },
  navigator_props: generate_coherent_navigator(),
  created_at: timestamp,
  invalidate_on_origin_change: true
}

Integration Points

The engine intercepts at critical junctures:

  • WebGL: getParameter(RENDERER), getParameter(VENDOR), shader compilation behavior
  • Canvas: getImageData() rasterization, pixel-perfect rendering output
  • AudioContext: sampleRate, channelCount, maxChannelCount
  • Navigator: User agent parsing, plugin enumeration, device memory spoofing

WebGL Renderer Spoofing

WebGL exposes the GPU renderer string through UNMASKED_RENDERER_WEBGL. This is trivial to read and instantly identifies your GPU model:

const canvas = document.createElement('canvas');
const gl = canvas.getContext('webgl');
const ext = gl.getExtension('WEBGL_debug_renderer_info');
const renderer = gl.getParameter(ext.UNMASKED_RENDERER_WEBGL);
// Returns: "ANGLE (Intel HD Graphics 630)"

Our approach: synthesize a plausible renderer string that matches the platform's constraints but remains constant per-origin.

function synthesize_renderer(profile, platform) {
  const vendors = platform === 'win32' 
    ? ['ANGLE (NVIDIA GeForce GTX 1080)', 'ANGLE (AMD Radeon RX 580)', ...]
    : ['Apple M1', 'Intel Iris Plus Graphics 640', ...];
  
  // Deterministic selection based on origin hash
  const idx = profile.origin_hash % vendors.length;
  return vendors[idx];
}
Timing Variance

GPU operations have measurable timing variance. We add controlled jitter (±2ms) to shader compilation times to prevent timing-based discrimination while staying within realistic bounds.

Canvas Fingerprinting Defense

Canvas fingerprinting exploits subtle differences in rendering engines and font rasterization:

const canvas = document.createElement('canvas');
const ctx = canvas.getContext('2d');
ctx.font = '16px Arial';
ctx.fillText('Browser Fingerprinting Test', 2, 15);
const data = canvas.toDataURL();
// Site hashes this; same browser = same hash = identified

We intercept at the pixel level, applying subtle but consistent modifications per-origin that break fingerprinting while maintaining visual fidelity:

void canvas_getImageData_hook(origin_hash) {
  uint8_t seed = (origin_hash >> 24) & 0xFF;
  for (pixel in image_data) {
    // Add deterministic noise: ±1 per channel
    // Same origin = same noise = survives intra-origin detection
    // Different origin = different noise = breaks cross-site tracking
    pixel.r += (seed + pixel.x) % 3 - 1;
    pixel.g += (seed + pixel.y) % 3 - 1;
    pixel.b += (seed + pixel.x + pixel.y) % 3 - 1;
  }
}

AudioContext Spoofing

AudioContext exposes system audio capabilities that often correlate with device type and OS:

const ctx = new AudioContext();
console.log(ctx.sampleRate);        // 48000 on most systems, 44100 on others
console.log(ctx.destination.maxChannelCount);  // 2, 6, 8...

We generate compatible audio properties based on the platform:

AudioContext* create_spoofed_context(origin_hash) {
  Platform platform = get_platform();
  AudioProfile profile = profiles[origin_hash % profiles.size()];
  
  uint32_t sample_rate = (platform == PLATFORM_WIN) 
    ? 48000 : 44100;  // Keep realistic per-platform
  
  uint8_t channels = platform_supports_surround()
    ? (origin_hash % 6) + 2  // 2-8 channels, deterministic
    : 2;
    
  return new AudioContext(sample_rate, channels);
}

Navigator Property Coherence

The navigator object leaks platform details through properties like hardwareConcurrency, deviceMemory, platform:

We generate a coherent navigator profile that remains consistent within an origin:

navigator.hardwareConcurrency = 8;     // Must match CPU reality
navigator.deviceMemory = 16;            // Must stay plausible (4, 8, 16, 32)
navigator.platform = "Win32";           // Must match OS
navigator.userAgent = "Mozilla/5.0..."; // Must parse consistently
Consistency Guarantee

All navigator properties are derived from a single origin_hash, ensuring that sites cannot find internal contradictions that expose our spoofing.

Defense Against Timing Attacks

Even with spoofed data, timing side-channels can leak information. GPU operations, memory access patterns, and cryptographic operations all have measurable variance.

We add jitter at predictable intervals without breaking real-time performance:

void add_timing_jitter(TimingContext ctx) {
  uint64_t jitter_seed = derive_jitter_seed(ctx.origin_hash);
  
  // Deterministic pseudo-randomness per origin
  // Never breaks functionality, only adds noise
  uint32_t jitter_ms = (jitter_seed % 3) + 1;  // ±1-3ms
  
  if (should_apply_jitter(ctx.operation_type)) {
    add_delay(jitter_ms);
  }
}

Cross-Origin Tracking Prevention

By ensuring each origin gets a unique profile while maintaining internal coherence:

  • Same-origin detection: ✓ Still works (deterministic per-origin data)
  • Cross-origin correlation: ✗ Blocked (different profiles per origin)
  • Intra-browser correlation: ✗ Impossible (no global fingerprint)
